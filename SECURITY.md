# Security Policy

## Supported Versions

We actively maintain and release security patches for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue in AgentGuard, please follow responsible disclosure practices.

**Please do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

1. **Email**: Send details to the repository owner via GitHub's private security advisory feature.
2. **GitHub Security Advisories**: Use the [Security tab](../../security/advisories/new) in this repository to submit a private advisory.

### What to Include

- A clear description of the vulnerability
- Steps to reproduce the issue
- The potential impact of the vulnerability
- Any suggested fixes or mitigations (optional)

## Response Timeline

- **Acknowledgement**: Within 48 hours
- **Status Update**: Within 7 days
- **Resolution**: We aim to address critical vulnerabilities within 30 days
