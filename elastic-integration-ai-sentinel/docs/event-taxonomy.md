# Event Taxonomy

`ai_sentinel.finding.type` is the primary taxonomy key for this Elastic integration. Producers emit one type per finding; the ingest pipeline normalizes ECS metadata and the SIEM rules query ECS plus `ai_sentinel.*` fields.

All examples are metadata-only. Do not emit prompt text, completion text, clipboard content, browsing history, decrypted traffic, secrets, or private file contents.

| Finding type | Description | Example source | Expected ECS categories | Expected risk level | Expected key fields | Expected SIEM use case |
|---|---|---|---|---|---|---|
| `ai_api_connection` | Process communicates with a known or suspected AI provider/API endpoint. | Firewall, EDR network event, proxy metadata, or scanner-produced connection inventory. | `event.category: network`, `event.type: connection` | Low for approved clients; medium/high for unknown or untrusted processes. | `destination.domain`, `destination.port`, `network.protocol`, `process.executable`, `ai_sentinel.ai.provider`, `ai_sentinel.ai.endpoint`, `ai_sentinel.allowed` | Detect unknown or unapproved AI API clients and trend AI provider usage. |
| `mcp_server` | MCP server configuration or runtime metadata is present. | MCP config file inventory or application config export. | `event.category: configuration`, `event.type: info` | Medium/high when shell or filesystem capabilities are exposed by an untrusted server. | `ai_sentinel.mcp.client.name`, `ai_sentinel.mcp.server.name`, `ai_sentinel.mcp.server.command`, `ai_sentinel.mcp.tools`, `ai_sentinel.mcp.capabilities` | Alert on untrusted MCP servers exposing dangerous tools. |
| `mcp_config_modified` | MCP configuration file was created or modified. | File integrity metadata or scanner-produced config diff summary. | `event.category: configuration`, `event.type: info` | Medium when new servers or expanded capabilities appear; high if dangerous tools are added. | `file.path`, `file.name`, `ai_sentinel.mcp.config.path`, `ai_sentinel.mcp.capabilities`, `ai_sentinel.risk.reasons` | Review new or changed MCP server configuration. |
| `browser_extension` | Browser extension has AI-related behavior or risky permissions. | Browser extension inventory metadata. | `event.category: configuration`, `event.type: info` | Low for approved narrow permissions; high for broad host access or native messaging. | `ai_sentinel.browser.name`, `ai_sentinel.extension.id`, `ai_sentinel.extension.permissions`, `ai_sentinel.extension.host_permissions` | Detect browser extension risk such as `all_urls` plus native messaging. |
| `startup_item` | AI tool, agent, or MCP bridge is configured for persistence. | Startup item inventory metadata. | `event.category: configuration`, `event.type: creation` | High when unapproved; critical only when combined with stronger dangerous behavior. | `ai_sentinel.startup.type`, `ai_sentinel.startup.name`, `ai_sentinel.startup.path`, `ai_sentinel.startup.command`, `process.executable` | Alert on AI agent persistence or unexpected startup registration. |
| `local_llm_service` | Local LLM service listener metadata is observed. | Listener inventory metadata or local service configuration. | `event.category: network`, `event.type: info` | Low when loopback-only and approved; high when bound to non-loopback addresses. | `destination.ip`, `destination.port`, `ai_sentinel.ai.local_service`, `ai_sentinel.ai.model_hint`, `process.executable` | Detect exposed local LLM services reachable by other hosts. |
| `suspicious_agent_process` | Agent-like process behavior requires review. | Process metadata summary from a future scanner or EDR enrichment. | `event.category: process`, `event.type: info` | Medium/high based on untrusted path, execution context, and combined capabilities. | `process.name`, `process.executable`, `process.command_line`, `ai_sentinel.risk.reasons`, `ai_sentinel.allowed` | Identify unapproved AI automation runtimes without using prompt content. |
| `ai_cyber_agent_activity` | General behavior-based AI cyber-agent activity. | Scanner-produced correlation of AI API, tool, shell, filesystem, or browser metadata. | `event.category: process`, `event.type: info` | High/critical when multiple strong dangerous capabilities combine. | `ai_sentinel.cyber_agent.name`, `ai_sentinel.cyber_agent.framework`, `ai_sentinel.cyber_agent.capabilities`, `ai_sentinel.cyber_agent.security_tools` | Catch broad untrusted cyber-agent behavior. |
| `ai_vulnerability_research_agent` | AI-assisted vulnerability research metadata. | Repository analysis metadata, tool invocation metadata, or normalized file/path labels. | `event.category: process`, `event.type: info` | Medium/high; critical only with execution, sensitive access, or exploit-like writes. | `ai_sentinel.cyber_agent.activity_type`, `ai_sentinel.cyber_agent.target_paths`, `ai_sentinel.cyber_agent.suspicious_keywords`, `ai_sentinel.cyber_agent.security_tools` | Review AI-assisted vulnerability research outside approved contexts. |
| `ai_sandbox_escape_research` | Sandbox, container, or isolation escape research indicators. | Metadata for accessed paths, commands, or research labels. | `event.category: process`, `event.type: info` | High for unapproved research; critical if paired with exploit-like writes or shell/filesystem access. | `ai_sentinel.cyber_agent.target_paths`, `ai_sentinel.cyber_agent.suspicious_keywords`, `process.executable` | Alert on unapproved sandbox escape research activity. |
| `ai_fuzzing_activity` | AI-assisted fuzzing metadata. | Fuzzer process metadata, corpus path metadata, or crash triage labels. | `event.category: process`, `event.type: info` | Low/medium for approved researchers; high when unapproved or run from untrusted paths. | `ai_sentinel.cyber_agent.security_tools`, `ai_sentinel.cyber_agent.target_paths`, `ai_sentinel.cyber_agent.activity_type` | Distinguish approved fuzzing from unexpected agent-driven fuzzing. |
| `ai_reverse_engineering_activity` | AI-assisted reverse engineering metadata. | Disassembler/debugger tool metadata or binary target metadata. | `event.category: process`, `event.type: info` | Medium/high depending on target sensitivity and approval state. | `ai_sentinel.cyber_agent.security_tools`, `file.path`, `process.executable`, `ai_sentinel.cyber_agent.suspicious_keywords` | Review AI-assisted reverse engineering on sensitive systems. |
| `ai_exploit_development_activity` | Exploit-like file or project metadata associated with an AI agent. | File write metadata, project metadata, or tool invocation metadata. | `event.category: process`, `event.type: creation` | High/critical when untrusted and paired with exploit-like writes or security tooling. | `file.name`, `file.path`, `ai_sentinel.cyber_agent.activity_type`, `ai_sentinel.cyber_agent.suspicious_keywords` | Alert on exploit-like synthetic PoC or payload development. |
| `ai_security_tool_mcp_server` | MCP server exposes security tools to an AI agent. | MCP configuration or runtime tool inventory. | `event.category: process`, `event.type: info` | High when untrusted; critical with shell/filesystem and broad target access. | `ai_sentinel.mcp.server.name`, `ai_sentinel.mcp.tools`, `ai_sentinel.cyber_agent.security_tools`, `ai_sentinel.cyber_agent.capabilities` | Detect AI agents with MCP access to security tooling. |
| `ai_agent_shell_tool_use` | AI agent has or uses shell execution capability. | MCP tool metadata, process metadata, or agent tool inventory. | `event.category: process`, `event.type: info` | High when untrusted; critical if combined with filesystem, browser, or sensitive paths. | `process.command_line`, `ai_sentinel.cyber_agent.capabilities`, `ai_sentinel.mcp.tools` | Alert on dangerous shell-capable AI agent activity. |
| `ai_agent_sensitive_repo_scan` | AI agent scans sensitive repository areas. | Repository path scan summary or code index metadata. | `event.category: file`, `event.type: info` | High for unapproved scans of auth, secrets, CI/CD, cloud, or token-adjacent paths. | `file.path`, `ai_sentinel.cyber_agent.target_paths`, `ai_sentinel.cyber_agent.codebase_scan_volume`, `ai_sentinel.cyber_agent.suspicious_keywords` | Detect source-code reconnaissance around sensitive components. |
| `ai_agent_mass_codebase_analysis` | AI agent performs large codebase analysis. | Code indexing or repository traversal summary. | `event.category: process`, `event.type: info` | Medium/high based on volume, scope, and approval status. | `ai_sentinel.cyber_agent.codebase_scan_volume`, `ai_sentinel.cyber_agent.target_paths`, `ai_sentinel.ai.provider`, `ai_sentinel.allowed` | Monitor large-scale AI codebase analysis and exfiltration-adjacent patterns. |

## Taxonomy principles

- Prefer behavior labels over prompt-derived labels.
- Use `ai_sentinel.cyber_agent.suspicious_keywords` only for normalized metadata keywords from filenames, commands, paths, or rule observations.
- Do not place raw prompts, completions, code contents, file contents, or secrets into taxonomy fields.
- Combine weak indicators with concrete behaviors before assigning high or critical risk.
