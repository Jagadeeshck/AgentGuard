# AgentGuard semantic conventions

AgentGuard defines a neutral `agentguard.*` semantic model for AI activity from many sources. It is backend-neutral and tool-neutral.

## Required fields

| Field | Meaning |
| --- | --- |
| `agentguard.schema.version` | AgentGuard schema version. |
| `agentguard.source.type` | Source adapter type: native OTEL, API gateway, browser metadata, MCP/runtime, endpoint sensor, or SaaS audit. |
| `agentguard.source.name` | Adapter or source integration name. |
| `agentguard.provider.name` | Generic provider name such as `openai`, `anthropic`, `google`, `microsoft`, `local`, or `unknown`. |
| `agentguard.provider.category` | Provider category such as `commercial_ai`, `coding_assistant`, `local_llm`, `framework`, or `mcp`. |
| `agentguard.activity.type` | Normalized activity type. |
| `agentguard.finding.id` | Stable finding/event identifier generated by AgentGuard or a source adapter. |
| `agentguard.finding.type` | Detection or event classification. |
| `agentguard.risk.level` | `none`, `low`, `medium`, `high`, or `critical`. |
| `agentguard.risk.score` | Integer risk score from 0 to 100. |
| `event.module` | Always `agentguard`. |
| `event.dataset` | `agentguard.findings`, `agentguard.otel`, or `agentguard.ai_activity`. |
| `observer.vendor` | Always `AgentGuard`. |
| `observer.product` | Always `AgentGuard`. |

## Recommended fields

Use session, request, model, application, user hash, team, tool, MCP, runtime, cost, token, duration, network, and privacy fields listed in the README. User identifiers should be hashed or otherwise pseudonymized where possible.

## Activity types

`ai_web_session`, `ai_api_request`, `ai_agent_execution`, `ai_tool_invocation`, `mcp_server_connection`, `mcp_tool_invocation`, `local_llm_service_detected`, `browser_ai_extension_detected`, `model_usage`, `policy_match`, `privacy_risk`, `cost_anomaly`, and `security_finding`.

## Privacy attributes

The default is metadata-only:

- `agentguard.privacy.prompt_capture_enabled=false`
- `agentguard.privacy.content_capture_enabled=false`
- `agentguard.privacy.redaction_status=metadata_only`
