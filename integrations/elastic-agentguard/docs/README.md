# AgentGuard AgentGuard integration

## About this integration
This integration ingests metadata-only findings generated by agentguard-sensor NDJSON output and normalizes them for Elastic Security workflows.

## Requirements
- Elastic Stack 8.13.0+
- Fleet Server and Fleet-managed Elastic Agent
- Access to endpoint filesystem paths where agentguard-sensor writes findings

## Setup
1. In Kibana, go to **Management > Integrations** and install **AgentGuard AgentGuard**.
2. Add the integration to an Elastic Agent policy.
3. Configure `paths` to the sensor output file (for example `/var/log/agentguard/findings.ndjson`).
4. Save and deploy the policy.

## Data streams
- `findings` (`logs-agentguard.findings-<namespace>`): AgentGuard findings events.

## Fields
Custom fields are under the `agentguard.*` namespace in `data_stream/findings/fields/fields.yml`.

## Privacy notes
This integration is intended for metadata-only events. Do not send prompt content, clipboard content, or browsing history. See `contracts/producer-consumer-contract-v1.md`.
